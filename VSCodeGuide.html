01 Introduction — What is VS Code?

Visual Studio Code (VS Code) is a free, open-source code editor developed by Microsoft. It runs on Windows, macOS, and Linux, and supports hundreds of programming languages out of the box. With its rich extension ecosystem, integrated terminal, debugger, and Git support, VS Code has become the most popular developer editor worldwide.

Key features include:

  • IntelliSense — smart code completions based on language semantics
  • Built-in Git — stage, commit, push, pull, and resolve conflicts without leaving the editor
  • Integrated Terminal — full shell access right in the editor
  • Debugger — set breakpoints, inspect variables, and step through code
  • Extensions — thousands of community-built plugins for languages, themes, and tools
  • Remote Development — connect to WSL, containers, or SSH servers
Analogy: VS Code is like a Swiss Army knife for developers — lightweight by default, but infinitely expandable through extensions. It replaces the need for multiple separate tools (editor, terminal, debugger, Git client).

02 Installation & First Launch Beginner

Download & Install

  • Linux (Debian/Ubuntu): Download .deb from code.visualstudio.com or use sudo snap install code --classic
  • Linux (Fedora): Download .rpm from code.visualstudio.com
  • macOS: Download .zip from code.visualstudio.com or brew install --cask visual-studio-code
  • Windows: Download the installer from code.visualstudio.com

Launch from Terminal

# Open VS Code in the current directory
code .

# Open a specific file
code index.js

# Open and wait for the window to close (useful for Git commit messages)
code --wait file.txt

# Open without a window (add to PATH during install on Windows/macOS)
code --helpCopy
💡 Tip: On macOS, after installing, open the Command Palette (Cmd+Shift+P), type Shell Command: Install 'code' command in PATH, and run it to enable the code terminal command.

First Launch Checklist

  • Open the Command Palette (Ctrl+Shift+P) and try typing commands
  • Toggle the Explorer sidebar (Ctrl+B)
  • Open the Extensions panel (Ctrl+Shift+X) and search for a theme
  • Open the Integrated Terminal (Ctrl+`)

💡 Tip: Type ? in the palette to see available special prefixes: > for commands, @ for symbols, : for line numbers, and # for file names.

Quick Open (Ctrl+P)

Press Ctrl+P and type part of a filename to navigate instantly. Add modifiers:

  • @ — go to a symbol within the current file
  • : — go to a specific line (e.g., :42)
  • # — search across all files for a symbol
  • > — open the Command Palette (same as Ctrl+Shift+P)

# Clear terminal
# Type 'clear' in the terminal or click trash icon

# Open file links in terminal output
# Ctrl+Click on a file:line in terminal outputCopy

Default Shell

You can change the default shell via the Command Palette: Terminal: Select Default Profile. Options include:

  • bash, zsh, fish (Linux/macOS)
  • PowerShell, Command Prompt, Git Bash (Windows)
  • WSL (Windows Subsystem for Linux)
Pro tip: You can open a terminal directly at a project root by right-clicking a folder in the Explorer and selecting Open in Integrated Terminal.

Popular Settings

{
  // Editor appearance
  "editor.fontSize": 14,
  "editor.fontFamily": "'Fira Code', 'Cascadia Code', 'JetBrains Mono', monospace",
  "editor.fontLigatures": true,
  "editor.lineHeight": 24,
  "editor.minimap.enabled": true,
  "editor.renderWhitespace": "boundary",
  "editor.bracketPairColorization.enabled": true,

  // Format on save
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",

  // Word wrap
  "editor.wordWrap": "on",

  // Auto save
  "files.autoSave": "onFocusChange",

  // Tab size
  "editor.tabSize": 2,
  "editor.insertSpaces": true,

  // Explorer
  "explorer.compactFolders": false,
  "explorer.confirmDragAndDrop": false,

  // Terminal
  "terminal.integrated.fontSize": 13,
  "terminal.integrated.defaultProfile.linux": "bash",

  // Git
  "git.enableSmartCommit": true,
  "git.autofetch": true
}Copy

Settings Layers

  • User Settings — apply to every project (global)
  • Workspace Settings — stored in .vscode/settings.json, shared with the team
  • Folder Settings — apply to a specific folder in a multi-root workspace
⚠ Workspace settings override user settings. This is useful for project-specific formatting rules.

09 Snippets Intermediate

Snippets let you insert repetitive code blocks quickly by typing a prefix and pressing Tab.

Built-in Snippets

VS Code comes with built-in snippets for many languages. For example, in JavaScript/TypeScript:

// Type 'log' and press Tab
console.log();

// Type 'fn' in JS/TS
function name() {

}

// Type 'for' → for loop
for (let index = 0; index < array.length; index++) {
  const element = array[index];
}

// Type 'clg' in JS
console.log()Copy

Create Your Own Snippets

// Open: Command Palette → Preferences: Configure User Snippets
// Choose a language or create a global one

{
  // Example: React functional component snippet
  "React Functional Component": {
    "prefix": "rfc",
    "body": [
      "import React from 'react';",
      "",
      "const ${1:ComponentName} = () => {",
      "  return (",
      "    <div>$0</div>",
      "  );",
      "};",
      "",
      "export default ${1:ComponentName};"
    ],
    "description": "React Functional Component"
  }
}Copy
💡 Tip: $1, $2 are tab stops. $0 is the final cursor position. ${1:default} creates a placeholder with default text. Wrap in ${1|a,b|} for a choice list.

10 Debugging Intermediate

VS Code has a built-in debugger that supports Node.js, Python, Java, C++, and more via extensions.

Getting Started

# Open Run and Debug panel
Ctrl+Shift+D

# Start debugging (F5)
F5

# Toggle breakpoint (click gutter or F9)
F9

# Step over
F10

# Step into
F11

# Step out
Shift+F11

# Continue (resume execution)
F5

# Stop debugging
Shift+F5

# Restart debugging
Ctrl+Shift+F5Copy

Launch Configuration

Create a .vscode/launch.json file. Here's an example for Node.js:

{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Launch Program",
      "skipFiles": ["<node_internals>/**"],
      "program": "${workspaceFolder}/index.js"
    },
    {
      "type": "node",
      "request": "attach",
      "name": "Attach to Process",
      "port": 9229
    }
  ]
}Copy
Variable reference: ${workspaceFolder} — current project root, ${file} — currently open file, ${lineNumber} — current line, ${env:VAR} — environment variable.

Debug Features

  • Watch — track variable expressions
  • Call Stack — see the execution path
  • Variables — inspect local, global, and closure variables
  • Logpoints — log messages without stopping (right-click gutter → Add Logpoint)
  • Conditional Breakpoints — break only when an expression is true

# Alt+Click to add a cursor wherever you click

# Undo last cursor action
Ctrl+U

# Select all occurrences of current word
Ctrl+Shift+L

# Cancel multi-cursor
EscCopy

Practical Workflows

  • Rename a variable — select the variable, press Ctrl+Shift+L, type the new name
  • Edit multiple lines — select lines, press Ctrl+Shift+L, edit all at once
  • Add commas/semicolons — Alt+Shift+I to add a cursor at the end of every line in a selection
  • Column selection — hold Shift+Alt and drag to select a rectangular block

14 Emmet Intermediate

Emmet lets you write HTML and CSS at lightning speed using CSS-like abbreviations. Built into VS Code for HTML, CSS, JSX, and more.

HTML Abbreviations

// Type and press Tab

// Basic structure
!             → <!DOCTYPE html>... (full HTML5 boilerplate)

// Elements
div           → <div></div>
p             → <p></p>
a             → <a href=""></a>
img           → <img src="" alt="" />

// Nesting
nav>ul>li   → <nav><ul><li></li></ul></nav>

// Siblings
div+p+bq     → <div></div><p></p><blockquote></blockquote>

// Multiplication
ul>li*5      → <ul><li>*5</li></ul>

// IDs and classes
div#header   → <div id="header"></div>
div.container → <div class="container"></div>

// Attributes
a[href="https://example.com"] → <a href="https://example.com"></a>

// Text
p{Hello}     → <p>Hello</p>

// Numbered items
ul>li.item$*3 → <ul><li class="item1"></li>...</ul>Copy

CSS Abbreviations

// Properties
m            → margin: ;
p            → padding: ;
c            → color: #000;
bg           → background: #000;
bgc          → background-color: #000;
ff           → font-family: ;
fs           → font-size: ;
fw           → font-weight: ;
ta:c         → text-align: center;
d:f          → display: flex;
jc:c         → justify-content: center;
ai:c         → align-items: center;
bdrs         → border-radius: ;

// Vendor prefixes
// Many are handled automaticallyCopy

15 Tasks & Build Systems Advanced

Tasks let you run tools like build scripts, linters, test runners, or any shell command directly from VS Code.

Create a Task

Create a .vscode/tasks.json file:

{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "Build Project",
      "type": "shell",
      "command": "npm run build",
      "group": {
        "kind": "build",
        "isDefault": true
      },
      "problemMatcher": ["$tsc"]
    },
    {
      "label": "Run Tests",
      "type": "shell",
      "command": "npm test",
      "group": "test"
    },
    {
      "label": "Lint",
      "type": "shell",
      "command": "npm run lint"
    }
  ]
}Copy

Run a Task

  • Ctrl+Shift+B — run the default build task
  • Ctrl+Shift+P → Tasks: Run Task — pick any defined task
  • Tasks output goes to the Terminal panel, with error markers parsed by problemMatcher

Problem Matchers

Problem matchers parse compiler/linter output and highlight issues inline. Built-in matchers include:

  • $tsc — TypeScript compiler
  • $eslint-compact — ESLint compact format
  • $eslint-stylish — ESLint stylish format
  • $gcc — GCC/G++
  • $jshint — JSHint
Tip: You can define custom problem matchers using regex patterns in tasks.json problemMatcher object.

16 Remote Development Advanced

With the Remote Development extension pack, VS Code can open folders on remote machines, containers, or WSL as if they were local.

Remote Options

Remote — SSH
Connect to any machine via SSH. Full dev experience with port forwarding.
Dev Containers
Develop inside Docker containers with full toolchain isolation.
Remote — WSL
Open Linux projects in WSL with native Windows VS Code UI.
Remote — Tunnels
Expose your local VS Code securely over the internet.

Quick Start: SSH Remote

# Install "Remote — SSH" extension
# Open Command Palette → Remote-SSH: Connect to Host

# Or open from terminal
code --remote ssh-remote+dev.example.com /path/to/project

# Add SSH hosts to your config (~/.ssh/config)
Host dev-server
  HostName 192.168.1.100
  User ubuntu
  IdentityFile ~/.ssh/id_ed25519Copy

Quick Start: Dev Containers

# Create .devcontainer/devcontainer.json in your project
{
  "name": "Node.js Dev",
  "image": "mcr.microsoft.com/devcontainers/javascript-node:18",
  "forwardPorts": [3000],
  "extensions": ["dbaeumer.vscode-eslint"],
  "postCreateCommand": "npm install"
}

# Open Command Palette
# → Dev Containers: Reopen in ContainerCopy
⚠ Remote Development requires the Remote Development extension pack from Microsoft. Dev Containers need Docker installed on the host.
